# Sentinel-F

Веб-интерфейс системы пожарной и охранной сигнализации с редактором карты объекта, мониторингом зон и датчиков, управлением статусами.

Построено на Vue 3 + Konva, без бэкенда — карта хранится в памяти, сессия в `localStorage`.

---

## Возможности

- **Три роли** с разным интерфейсом:
  - **Инженер** — рисует карту объекта, размещает зоны и датчики, настраивает свойства
  - **Дежурный** — видит зоны и статусы, переключает их (норма / тревога), сбрасывает сработки
  - **Пользователь** — только просмотр карты и текущих статусов
- **Зоны со статусами**: зелёная заливка — норма, красная (с пульсацией) — сработка
- **Чёрный периметр** у комнат по умолчанию, светлые «стены» отдельным инструментом
- **Привязка** при рисовании и перетаскивании:
  - к узлам сетки (24 px)
  - к вершинам и центрам существующих фигур (порог 10 px)
  - маркер привязки на канвасе (зелёный — вершина, синий — сетка)
  - `Alt` — временно отключить
- **Датчики 4 типов**: пожарный (F), дыма (S), движения (M), двери (D); ставятся одним кликом, имеют свой статус
- **Контекстное меню** (ПКМ) с действиями для каждой роли
- **Шорткаты**: V/H/R/C/L/D/T для инструментов, Ctrl+D — дублировать, Del — удалить, F — по размеру, 0 — сброс вида
- **Экспорт карты** в JSON

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # production-сборка в dist/
npm run preview  # локальный просмотр сборки
```

Требуется Node 24.x.

## Модель данных

Каждая фигура имеет поля `id`, `kind`, `type` плюс специфичные:

| `kind` | `type` | Назначение | Поля |
|--------|--------|------------|------|
| `zone` | `rect`, `circle` | Охраняемая/контролируемая зона | `status`, `sensorKind`, `name` |
| `sensor` | `sensor` | Датчик в точке | `status`, `sensorKind`, `name` |
| `wall` | `line` | Стена / периметр | `points`, `stroke` |
| `label` | `text` | Текстовая метка | `text`, `fontSize`, `fill` |

Статус: `'ok'` или `'alarm'`. Цветовая схема в `STATUS` (`src/constants/editor.js`).

## Стек

- **Vue 3** (Composition API, `<script setup>`)
- **Konva 10** для канваса
- **Vite 8** для сборки
- Без роутера, без стейт-менеджера, без бэкенда — вся логика в одном composable

---

## Что не реализовано

- Бэкенд: карта не сохраняется между сессиями (только экспорт в JSON)
- Реальное подключение к датчикам / контроллерам сигнализации
- История событий и журнал сработок
- Аутентификация на сервере (учётки сейчас захардкожены на клиенте — это демо)
- Привязка к серединам отрезков стен
- Многопользовательская синхронизация
